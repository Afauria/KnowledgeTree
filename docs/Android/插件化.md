# 插件化和热修复

插件化和热修复都是通过自定义类加载器实现。

* 热修复是为了修复原有类的缺陷，所以要先加载新的类，替代原有类，类名相同。
* 插件化是为了增加新功能或新资源，所以不需要修改类加载顺序。实现更简单。

另外有的热修复框架还支持通过Diff算法对比新旧版本，生成差分包。
